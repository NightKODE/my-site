Lesson 1
Структура сайта.

L1. Структура сайта.

  • Header - шапка сайта.

    Тут находится основная навигация сайта, поиск, ссилки, логотип и т.д.

  • Main - тело страницы.

    Тут находится основной контент разделенный другими семантическими тегами - Section и Article.

  • Footer - "подвал" сайта.

    Тут находятся различные ссылки на различные ресурсы, контактные данные, дополнительная информация и прочее.

  • Nav - набор ссылок.
  • Section - семантические единые разделы страницы.
  • Article - логически обособленнй блок.

Lesson 2
Теги текстовых элементов.

L2. Теги текстовых элементов.

  • h1, h2, h3, h4, h5, h6 - блочные теги предназначены для указания заголовков. Заголовков в html есть 6 уровней, от h1 (первый уровень) до h6 (последний уровень), каждый визуально выделен жирным шрифтом и его соответствующим размером.

    Заголовки должны описывать содержание, это положительно влияет на оптимизацию сайта. Хорошей практикой считается указывать только один заголовок первого уровня h1, остальных может быть несколько. Заголовки должны указываться по иерархии, на первом уровне заголовок h1 дальше подзаголовок h2 и так далее соответствующим размером.

  • p - абзац. Блочный тег, разделяет текст на отдельные абзацы, в конце добавляется отступ.
  • q - отображает текст как цитату, обрамляет его кавычками.
  • blockquote - отображает текст как цитату. По сравнению с тегом используется для длинных цитат, оформляется отступами.
  • cite - выделяет цитату на материал из другого источника, обычно отображается курсивным шрифтом.

Lesson 3
Теги списков.

L3. Теги списков.

  • ul - маркированный список. Каждый элемент списка оборачивается тегами li
  • ol - нумерованный список. Каждый элемент списка оборачивается тегами li
  • dl - список терминов. Внутри используются теги: dt - название термина, dd - описание термина

Lesson 4
Вставка изображения.

L4. Вставка изображения.

  • img - встраивает изображение в документ.

    Атрибут src обязателен и содержит путь к изображению, которое вы хотите встроить в документ. Атрибут alt содержит текстовое описание изображения, которое не обязательно, но невероятно полезно для доступности — программы чтения с экрана читают это описание своим пользователям, так они знают какое изображение показано, и так же оно отображается на странице, если изображение не может быть загружено по какой-либо причине.

  • picture - служит контейнером для одного или более элементов source и одного элемента img для обеспечения оптимальной версии изображения для различных размеров экрана.

    Браузер рассмотрит каждый из дочерних элементов source и выберет один, соответствующий лучшему совпадению; если совпадений среди элементов source найдено не будет, то будет выбран файл, указанный атрибутом src элемента img . Затем выбранное изображение отображается в пространстве, занятом элементом img.

  • figure - Элемент иллюстрации с необязательной подписью

    Представляет самостоятельный контент, часто с подписью (заголовком), которая указывается с помощью элемента figcaption. Диаграмма и её подпись представляет собой единое целое.

Lesson 5
Теги видео и аудио.

L5. Теги видео и аудио.

  • video - используется для встраивания видео контента в документ.

    Видео элемент может содержать один или несколько источников видео. Чтобы указать источник видео, необходимо использовать атрибут src или элемент source браузер сам определит наиболее подходящий источник.

  • audio - используется для встраивания звукового контента в документ.

    Он может содержать один или более источников аудио, представленных с помощью атрибута src или элемента source – браузер выберет один наиболее подходящий. Он так же может предназначаться для потокового мультимедиа, используя интерфейс MediaStream.

Атрибуты для video и audio.

  • controls - атрибут плеера (по умолчанию используется плеер браузера).
  • loop - зацикливание воспроизведения.
  • autoplay - автоматическое воспроизведение.
  • muted - отключение звука.
  • preload="none" - предзагрузка файла.

    none - Не загружать аудиофайл.
    metadata - Загрузить только служебную информацию (продолжительность звучания и др.).
    auto - Загрузить аудиофайл целиком при загрузке страницы.

  • width="480" - атрибут ширины блока с видео.
  • poster="..//" - атрибут для вставки превью для видео.

Lesson 6
Ссылки и кнопки.

L6. Ссылки и кнопки.

  • a - определяет гиперссылку для перехода на определённое место на странице или на другую страницу в Интернете.
  • button - создаёт кликабельную кнопку, которая может быть использована в формах или в любом другом месте документа, который требует простой, стандартной кнопки.

Атрибуты для a и button.

  • href - Задает адрес документа, на который следует перейти.
  • target - Определяет тип переходя между адресами.

    _blank - Загружает страницу в новое окно браузера.
    _self - Загружает страницу в текущее окно.
    _parent - Загружает страницу во фрейм-родитель, если фреймов нет, то это значение работает как _self.
    _top - Отменяет все фреймы и загружает страницу в полном окне браузера, если фреймов нет, то это значение работает как _self.

  • type - oпределяет тип кнопки, который устанавливает ее поведение в форме. По внешнему виду кнопки разного типа никак не различаются, но у каждой такой кнопки свои функции.

    button - Обычная кнопка.
    reset - Кнопка для очистки введенных данных формы и возвращения значений в первоначальное состояние.
    submit - Кнопка для отправки данных формы на сервер.

L7. Теги таблицы.

  • thead - предназначен для хранения одной или нескольких строк, которые представлены вверху таблицы.

  • td - предназначен для создания одной ячейки таблицы.
  • tbody - предназначен для хранения одной или нескольких строк таблицы.
  • tfootпредназначен для хранения одной или нескольких строк, которые представлены внизу таблицы.

SVG.

L12. SVG

svg - Scalable Vector Graphics - «масштабируемая векторная графика» — это вид графики, который создают с помощью математического описания геометрических примитивов (линий, кругов, эллипсов, прямоугольников, кривых и так далее), которые и образуют все детали будущего изображения. В этом формате хранится не сама картинка, а инструкции для её построения по точкам и кривым. Они написаны на языке разметки SVG, расширяющем XML.
    Достоинства:
  • масштабирование
  • возможность открывать как HTML или CSS и менять в любом редакторе кода
  • вес намного меньше растровых изображений
    • Недостатки:
  • не поддерживается очень старыми браузерами
  • не используется для фотографий

Внедрение в html

Для удобства использования svg можно создать общий контейнер(изначально задав ему стиль: display: none;) со всеми изображениями. Там каждое изображение описать в тег simbol(ему необходимо передать изначально заданный атрибут viewBox и id ), а уже оттуда использовать их при помощи тега use(в него нужно вписать содержимое исходного svg) и атрибута xlink:href=""

Работа с svg

Для работы с svg можно использовать css, однако для изменения цвета самого вектора нужно использовать свойтво fill(иногда вектор состоит из разных составных частей, для них нужно использовать fill отдельно для каждого элемента).

При изменении размера вектора желательно сохранять те же пропорции.

jQuery — это бесплатная библиотека языка JavaScript для фронтенд-разработки.
У jQuery собственный синтаксис, который содержит фрагменты кода JavaScript, выраженные короткими командами. В библиотеке предусмотрены команды для решения типовых задач во фронтенд-разработке.

L4. События в jQuery

События - это любое деуствие пользователя, а иногда и браузера, будь то клик мышки, прокрутка страници или ввод текста в инпут.

jQuery способен обрабатывать эти события и запускать необходимые скрипты или функции.
Как происходит обработка событий - при клике на какой-либо элемент происходит погружение в DOM-дерево начиная с родительского элемента, тем самым, родительские элементы участвуют в обработке события. Затем событие "всплывает" обратно, давая родительским элементам второй шанс на обработку события.

События


Базовые


  • .on() - Универсальный метод для установки обработчиков событий на выбранные элементы страницы.
  • .off() - Удаляет обработчики, установленные с помощь .on().
  • .bind() - Устанавливает обработчик события на выбранные элементы страницы. Обработчик не сработает на элементах, появившихся после его установки.
  • .live() - Устанавливает обработчик события на выбранные элементы страницы. Обработчик сработает и на элементах, появившихся после его установки.
  • .delegate() - Устанавливает обработчик события на выбранные элементы страницы. Элементы выбираются с помощью уточняющего селектора. Обработчик будет действовать и на элементах, появившихся после его установки.
  • .one() - Устанавливает обработчик события на выбранные элементы страницы, который сработает только по одному разу, на каждом из элементов.
  • .unbind() - Удаляет обработчик событий у выбранных элементов.
  • .die() - Удаляет обработчик событий, который был установлен с помощью live().
  • .undelegate() - Удаляет обработчик событий, который был установлен с помощью delegate().
  • .trigger() - Выполняет указанное событие и запускает его обработчик.
  • .triggerHandler() - Запускает обработчик указанного события, без его выполнения.
  • jQuery.proxy() - По заданной функции, создает другую, внутри которой переменная this будет равна заданному значению.
  • event - Объект, содержащий данные о текущем событии. Передается всем обработчикам событий.

События мыши


  • .click() - Устанавливает обработчик "клика" мышью по элементу, либо, запускает это событие.
  • .dblclick() - Устанавливает обработчик двойного "клика" мышью по элементу, либо, запускает это событие.
  • .hover() - Устанавливает обработчик двух событий: появления/исчезновения курсора над элементом.
  • .mousedown() - Устанавливает обработчик нажатия кнопки мыши, либо, запускает это событие.
  • .mouseup() - Устанавливает обработчик поднятия кнопки мыши, либо, запускает это событие.
  • .mouseenter() - Устанавливает обработчик появления курсора в области элемента, либо, запускает это событие. Появление этого события, отработано лучше, чем стандартного mouseover.
  • .mouseleave() - Устанавливает обработчик выхода курсора из области элемента, либо, запускает это событие. Появление этого события, отработано лучше, чем стандартного mouseout.
  • .mousemove() - Устанавливает обработчик движения курсора в области элемента, либо, запускает это событие.
  • .mouseout() - Устанавливает обработчик выхода курсора из области элемента, либо, запускает это событие.
  • .mouseover() - Устанавливает обработчик появления курсора в области элемента, либо, запускает это событие.
  • .toggle() - Поочередно выполняет одну из двух или более заданных функций, в ответ на "клик" по элементу.

События клавиатуры


  • .keydown() - Устанавливает обработчик перехода клавиши клавиатуры в нажатое состояние, либо, запускает это событие.
  • .keyup() - Устанавливает обработчик возвращение клавиши клавиатуры в ненажатое состояние, либо, запускает это событие.
  • .keypress() - Устанавливает обработчик ввода символа с клавиатуры, либо, запускает это событие.

События формы


  • .focus() - Устанавливает обработчик получения фокуса, либо, запускает это событие.
  • .blur() - Устанавливает обработчик потери фокуса, либо, запускает это событие.
  • .focusin() - Устанавливает обработчик получения фокуса самим элементом или одним из его дочерних.
  • .focusout() - Устанавливает обработчик потери фокуса самим элементом или одним из его дочерних.
  • .select() - Устанавливает обработчик выделения текста, либо, запускает это событие.
  • .submit() - Устанавливает обработчик отправки формы, либо, запускает это событие.
  • .change() - Устанавливает обработчик изменения элемента формы, либо, запускает это событие.

События загрузки страницы


  • .ready() - Устанавливает обработчик готовности дерева DOM.
  • .load() - Устанавливает обработчик завершения загрузки элемента.
  • .unload() - Устанавливает обработчик ухода со страницы (при переходе по ссылке, закрытии браузера и.т.д.).

События браузера


  • .error() - Устанавливает обработчик ошибки при загрузке элементов (например отсутствие необходимой картинки на сервере).
  • .resize() - Устанавливает обработчик изменения размеров окна браузера, либо, запускает это событие.
  • .scroll() - Устанавливает обработчик "прокрутки" элементов документа, либо, запускает это событие.

L3. Единицы измерения.

  • 1. vh и vw

    Эти величины определяют процентное отбражение элемента на экране вьюпорта или девайса.
    1vh и 1vw равны одному проценту от вьюпорта.

  • 2. vmin и vmax.

    vmin - вычисления опираяются на минимальную сторону вьюпорта. Та, что меньше и будет влиять на элемент для которго установлено свойство vmin. Нижуе кубик с примером.

    Ширина данного блока зависит от меньшей стороны вьюпорта
    и его ширина будет 50% от нее.
    Ширина данного блока зависит от большей стороны вьюпорта и его ширинна будет 50% от нее.
  • 3. em и rem.

    em - учитывает характеристики родительского элемента, обнуляещего файла или параметров браузера и умножает его на введенное количество.
    rem - учитывает характеристики только самого главного элемента (html) и умножает его на введенное количество.

    Каждой строчке задан класс со стилем - font-size: 1.5em;

    • Working directory - рабочая директория.

    • Staging area - индекс.

      В эту область добавляются файлы перед сохранением.

      git status - показывает изменения в индексе которые были подготовлены для сохранения (коммита).

    • Repository - репозиторий (папка.git).

      Здесь сохраняются все версии файлов проекта.

      Так же тут сохраняются коммиты (указатели на определенные версии проекта).

      На примере папки Objekts (в ней сохраняются все объекты репозитория).

      Папки появляются после создания объектов gitом.

      При изменении файла, в репозитории (папка Objects) сохранится два разных файла (первая версия и вторая).

    Lesson 4.
    Рабочий процесс GIT.

    L4. Рабочий процесс GIT.

    • 1. Создается папка с файлами - проект.
      В нее добавлять файлы можно традиционным и не только способами.
    • 2. В терминале прописывается команда git init.
      Она нужно для инициализации репозитория GIT и создания скрытой папки.git.
    • 3. В индекс файлы добавляются командой - git add “name”
      После команды прописывается имя файла, если поставить точку, то добавятся все модифицированные файлы и созданные.
    • 4. После добавления изменений в индекс можно сохранить текущую версию проекта в репозиторий командой - git commit -m "messege".
    • 5. Командой git checkout можно перейти к определенной версии проекта.
      Из репозитория в рабочую директорию перезаписываются файлы, либо добавляются, либо удаляются, тем самым проект будет состоять из нужных версий файлов.

    Статусы отслеживания файллов.

    L5. Статусы отслеживания файллов.

    • Untracked - неотслеживаемый.

      Новые файлы, добавленные в рабочую директорию имеют статус "неотслеживаемые” т.к. еще не находятся в репозитории. Этот статус имеют все новые файлы изначально.

    • Staged - подготовленный.

      Файлы, добавленные в индекс для сохранения командой git add, имеют статус “подготовленные”

    • Unmpdified - немодифицированный.

      Файлы, сохраненные в репозиторий командой
      git commit, имеют статус “немодифицированный”

    • Modified - модифицированный.

      Файлы (находящиеся в РД, индексе и репозитории), взятые для каких-либо изменений, имеют статус модифицированные. Они могут в дальнейшем иметь статус “подготовленные” и “немодифицированные”.

    Удаленный репозиторий GitHub.

    L7. Удаленный репозиторий GitHub.

    Удаленный репозиторий позволяет хранить проект удаленно.

    Удаленные репозитории бывают публичные и частные. публичные репозитории могут скачивать все без ограничений, но загружать изменения в него только с личным доступом. Личный доступ дает админ репозитория ( владелец на GitHub ). С частным репозиторием работать можно только с личным доступом.

    Как работать с удаленным репозиторием.

    • 1. Сначала уделенный репозиторий скачивается на компьютер командой git clone "url".

      При скачивании git создает связь между локальным и удаленным репозиториями, при этом уделенному репозиторию задается имя по умолчанию - origin

    • Далее необходимо подключить удаленный репозиторий к локальному командой
      git remote add origin "url" ( имя удаленного репозитория заданное на GItHub при его создании ).

      После этого на удаленный репозиторий можно загружать какие-лмбо изменения, но связь с ветками еще не устанавлена, это нужно делать далее.

    • Теперь локальную ветку (например “new-feature”) необходимо связать с удаленной веткой (например “main”, как правило связывают ветки с одинаковым названием) командой git push –u origin "branch".

      После создания связи между ветками загружать обновления можно обычной командой git push.

    • После проделанных действий загружать и скачивать какие-либо изменения в локальной ветке можно командами git push и git pull соответственно.

    Переменные.

    L3. Переменные.

    Переменные дают возможность повторного доступа к значениям, которые присвоены к этим переменным.

    Как правильно называть переменные.

    Имена переменных можно выбирать любое, но лучше использовать следующие три варианта:
    • 1. PascalCase - (первое слово названия переменной начинается с заглавной буквы, как и второе слово. Слова пишутся слитно) используется чтобы называть типы и классы в JS.
    • 2. DB_PASSWORD - значение такой переменной известно до запуска приложения и не меняются. (все слова пишутся капсом и между словами используется нижнее подчеркивание.)
    • 3. сamelCase - используется для всех остальных переменных, они используются чаще всего. (первое слово всегда в нижнем регистре, следующее слово с заглавной буквы, слова пишутся слитно)
    Название переменной должно быть понятным и отображать значение переменной.

    Объявление переменных.

    В JS есть три ключевых слова, с помощью которых объявляются переменные:
    • 1. let - let a объявление переменной с названием а, с помощью let мы даем инструкцию интерпритатору JS создать новую переменную с именем а. Присвоить значение этой переменной можно выражением присваивания.

      При использовании let значение переменной можно присвоить сразу, а позже можно присвоить другое значение.

    • 2. const - const c = 10 это объявление переменной с именем с и присваивание этой переменной значения 10 с помощью оператора присвоения.

      При использовании const присвоить другое значение переменной нельзя.

    • 3. var – не рекомендуется использовать

    Правила работы с переменными.

    • 1. Все переменные необходимо объявлять перед их использованием.
    • 2. При объявлении переменных использовать const везде, где только возможно.
    • 3. Внутри функции не изменять переменные из внешних областей видимости.

      Для этого лучше объявлять переменные внутри функции.

    L12. Мутация в JavaScript.

    При примитивных значениях переменных, копирование происходит по принципу copy by Value, то есть, копируется непосредственно само значение.
    При значениях типа объект, копирование происходит по принципу copy by Referens, то есть, копируется ссылка на объект, что позволяет изменять (мутировать) объект.

    Избежание мутаций.

    • Первый способ:

      Использовать метод assign объекта Object (этот объект является частью глобального объекта. Он так же является и типом и классом в JavaScript).

      При помощи такого синтаксиса можно создать новый объект на основе уже существующего. Таким образом создается новый объект с такими же свойствами, как у предыдущего, но с другой ссылкой на этот объект.

      Если же в предыдущем объекте имеются вложенные объекты, то ссылки на них сохранятся! Таким образом можно избежать мутации только корневых свойств.

    • Второй способ:

      Использовать оператор ..., он же оператор sprat (оператор разделения). Этот оператор разделяет объект на свойства, а записанный вместе с ним объект в фигурных скобках создает новую ссылку на объект.

      В это случае ситуация с вложенными объектами такая же, как и в первом.

    • Третий способ:

      Можно использовать двойную конвертацию объекта через JSON. Сначала объект конвертируется в строку, а потом конвертируется обратно в объект, создавая новые ссылки даже на вложенные объекты.

    Функции.

    L13. Функции.

    Функция - блок кода, который можно выполнять многократно.
    Если в коде часто встречаются одни и те же блоки кода, то такой код написан неоптимально. Чтобы избежать повторения, можно использовать функцию.
    После строки возвращения результата фукция прекращает работу.
    Функция возвращает undefiend, если нет инструкции return

    Объявление функции

    Функцию можно объявить несколькими способами:
    • При помощи самого слова function - function declaration:
      Особенностью этой функции является то, что ее можно использовать до ее фактического объявления. Эта механика называется поднятие или hosting.

      Перезапись


      Если ниже функцияя будет преписана, то при вызове функции будет выдаваться результат последней записанной.
    • При помощи ключевого слова const, как объявление переменной - function expression :
      В этом случае функция объявляется как переменная, и переписать ее уже нельзя, однако, если объявить функцию при помощи ключевого слова let, то переписать фунцию можно.

      Поднятие


      Такую функцию использовать до ее объявления нельзя.
    • Стрелочная функция - arrow function:
      Такой синтаксис набирает популярность, т.к. он короче, чем первые два.

      Поднятие


      Использовать эту функцию до ее объявления так же нельзя.

    Вызов функции

    При объявлении функции будет создана новая переменная, значением котрой является функция, после чего ее можно вызвать.
    При объявлении функции в скобках находятся парметры, являющиеся переменными, которые действуют в функции.
    При вызове функции в тех же скобках указываются уже аргументы. Они мгут меняться каждый раз при вызове этой функции.
    • 1. Параметрам а и b присваиваются значения 10 и 3.
    • 2. Объявляется переменная с.
    • 3. Значение а увеличивается на 1.
    • 4. Сумма значений а и b присваевается с.
    • 5. Возвращается значение с.

    Передача значения по ссылке

    В данном примере происходит мутация объекта personOne, поскольку при указании его в аргументе функции, происходит передача значения по ссылке, а не создается новый объект.
    Внутри функции мутировать внешние объекты не рекомендуется. Чтобы этого избежать, можно внутри функции создать копию объекта.

    Колбэк функции

    Callback function - это функция, короая вызывается внутри другой функции.
    setTimeout - это встроенная функция глобадьного объекта Objekt, и означет она задержку перед выполнением функции.

    Правила работы с функциями.

    • 1. Называть функции исходя их из выполняемых задач.
    • 2. Одна функция должна выполнять одну задачу.
    • 3. Не рекомендуется изменять внешние относительно функции переменные.

    L14. Области видимости

    Обдасть видимости определяется границы действия переменной.
    Есть глобальные переменные, которые объявлены в глобальной обласьт видимости, а есть локальный переменные, которые объявлены в локальной области видимости. Если попробовать получить лоступ к переменной b в локальной области видимости 1, то мы увидим результат переменной b, объявленной в глобальной области видимости.
    В данном примере функция является локальной областью видимости, в которой была создана еще одна переменная с именем b. Так же внутри этой функции было присвоено значение переменной a = true, однако присвоено оно пременной a из глобальной области видимости.
    В данном примере создается переменная myFn, внутри нее создается перменная innerFn, и если попытаться вызвать функцию innerFn, то произойдет ошибка, т.к. innerFn существиет только внутри функции myFn.
    Если в локальной области видимости присваяивать значения необъявленным переменным, то по завершению кода эти переменные будут созданы овтоматически. Так делать крайне не рекомендуется.

    Lesson 15
    Массивы.

    L15. Массивы.

    Массивы array позволяют хранить данные упорядоченным списком чего-либо, что предоставляет методы управления порядком элементов. Они могут хранить абсолютно разные значения.

    Формат записи массивов:

    При объявлении массива JS создаёт свойства от 0 до бесконечности (в зависимости от колличества объектов массива), значением этих свойств будут непосредственно сами объекты в порядке их объявления, и свойство lenght, а его значение зависит от общего колличества объектов.

    Массивы VS объекты

    Если попробовать объявить переменную с такими же свойствами как у массива, то на выходе в консоли мы получим такие же свойства за исключением __proto__. Однако по факту они абсолютно разные объекты.
    Поскольку свойство массива lenght создается самим JS то его значение будет всегда актуальным(если удалить или добавить новый объект в массив). В объекте же это свойство (при тех же манипуляциях) останется таким, каким было задано при объявлении объекта.

    Чтение свойств объекта

    Чтобы получить доступ к оъекты массива нужно при вызове этого массива написать в конце квадратные скобки и внутри написать индекс требуемого объекта от 0!. Таким же способом можно добавлять новые объекты в массив или менять существующие на новые.

Так, например, если создать файл index.html и в первой строчке файла поставить “!” знак и нажать на “Tab”, то у вас встроится следующий код:
      Это пустой шаблон html страницы. После встраивания данного сниппета вы можете менять его так, как Вам это нужно.
    • Как это работает?
      Сниппеты работают по принципу Emmet. Если вы когда-либо писали сайты или создавали приложения в Visual Studio, то, наверное, знаете про Emmet. Это набор встроенных или установленных плагинов, которые ускоряют разработку путем написания меньшего объема кода. По сути, то же, что будем делать и мы.       Нажмем Tab и преобразуем в...
    • Собственные сниппеты.
      После того, как мы разобрались, что такое сниппеты и какие они бывают по умолчанию, время разобраться с тем, как создать свой собственный. Чтобы создать свой сниппет в Visual Studio Code, необходимо перейти во вкладку “File” → “Preferences” → “User Snippets”.
      После этого в верху окна откроется окно выбора / создания сниппета:
      Здесь до горизонтальной разделительной черты находятся ваши собственные сниппеты. Далее есть пункт, который создает Глобальный сниппет «New Global Snippets file…», он может быть вызван и применен в любом проекте и на любом языке. Также можно создать локальный сниппет для конкретного проекта: «New Snippets file for ‘Project_name’…».     Все сниппеты создаются одинаково. Рассмотрим пример создания глобального сниппета.
    • 1. Перейдем во вкладку “File” → “Preferences” → “User Snippets”
    • 2. Выберем «New Global Snippets file…»
    • 3. Введем в появившемся окне наименование сниппета: «Test-snippet»
    • 4. Перед нами откроется окно с закомментированным текстом.
    • 5. Создаем и заполняем свой json фрагмент, сохраняем и все. Ваш сниппет готов

    Раскомментируем все, что находится ниже слова «Example:».

    «body» — это тело сниппета, здесь хранится то, что напечатается после нажатия на кнопку “Tab”.
